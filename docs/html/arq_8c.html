◆ ack_timeout_s_for_mode()

static int ack_timeout_s_for_mode ( int  mode)
static

Return ACK deadline budget in seconds for payload mode.

Rationale:

  • Timeout must cover peer decode + control reply scheduling + return airtime.
  • It is set above one slot length for each payload mode so one delayed cycle does not immediately trigger a retry/disconnect.
  • Mapping keeps relative ordering by payload burden: DATAC1 (largest frame) > DATAC3 > DATAC4 > control/default.
Parameters
modeFreeDV mode.
Returns
ACK timeout in seconds.

◆ mode_slot_len_s()

static int mode_slot_len_s ( int  mode)
static

Return per-mode slot timing budget in seconds.

Rationale:

  • Values are intentionally larger than pure modem airtime to absorb RX/TX turn switching, channel guard, decoder latency, and scheduler jitter.
  • Slower payload modes (larger on-air frame time) get longer slots: DATAC1 (510B) > DATAC3 (126B) > DATAC4 (54B) > control modes.
  • Control-only modes stay short to keep connect/turn/ack exchanges responsive.
Parameters
modeFreeDV mode.
Returns
Slot budget in seconds.
